# Public Disclosure Boundary

This repository is designed to be safe to share with SparkClaw and early
reviewers.

## Published

- High-level product and architecture description
- Public-safe technical capability narrative
- Simplified context runtime example
- Simplified overclaim / false-claim guard example
- AI-agent operating model
- Validation snapshot numbers from the private workspace

## Not Published

- Full mobile app source code
- Full Creator Tool source code
- Full Supabase, R2, Worker, and streaming implementation
- Private source audio, stems, mix sessions, or unreleased work assets
- Internal prompt history that exposes commercial strategy or private operating
  decisions
- Raw location traces, sensor logs, mic input, weather payloads, participant
  data, device identifiers, private tester notes, or incident details
- Secrets, tokens, signed URLs, bucket names, endpoint values, project refs, or
  cloud account details

## Why Not Publish Everything?

Aural Reality is a location-aware listening product. Publicly exposing every
internal artifact could leak:

- product strategy before validation;
- copyrighted or private sound material;
- unreleased technical gates;
- internal safety and incident handling notes;
- live infrastructure details that should never be public.

The goal of this repository is therefore narrower:

> Show enough technical substance to prove the work, while protecting the
> product, creators, testers, and future listeners.

## Claim Boundary

This repository does **not** claim:

- public launch;
- `launch_ready`;
- catalog count;
- listener streaming approval;
- native playback acceptance;
- physical-device evidence acceptance;
- field-test acceptance;
- legal / store / privacy finality;
- live backend readiness;
- production token issuance.

All such milestones require separate hard evidence in the private workspace.
